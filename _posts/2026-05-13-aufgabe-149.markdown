Bevor man hier loslegen konnte, musste man sich erstmal mit der "lagged Fibonacci random sequence" herumschlagen, um das quadratische Gitter überhaupt mit den richtigen Einträgen zu füllen. Schon ein wenig nervig, aber irgendwie auch lustig. Lesen wir doch mal die Inputs ein und schauen uns die Struktur einer möglichen Lösung in der main()-Routine an:

Der erste Schritt besteht darin, die Funktionsweise der "Zufalls"-Sequenzen zu verstehen. Wir bekommen unsere Einträge \\( s_i = a_{f_i} + b_{g_i} \\) als Summen zweier bestimmter Zahlen aus zwei gegebenen Vektoren (die habe ich passend "as" und "bs" benannt). An welchen Indizes wir die Summanden rausziehen müssen, sagen uns die beiden Folgen 

$$
\left( f_i \right)_{0\leq i < N^2} \quad \text{und} \quad \left( g_j \right)_{0 \leq j < N^2}
$$

die beide eine Fibonacci-artige Rekursionsrelation der Ordnung 5 erfüllen (die ersten nötigen Folgenglieder werden ebenfalls gegeben). 

In einer Funktion fill_rows() rechnen wir diese Folgenglieder Schritt für Schritt aus, berechnen sofort \\( s_i \\) und schreiben diesen Eintrag an die richtige Stelle ins Grid. Wenn wir damit fertig sind, schreiben wir alle Subarrays, die wir im Folgenden betrachten müssen in die dafür vorgesehenen Strukturen. Ich lasse fill_cols() und Teile der anderen Funktionen mal weg, das kann man sich überlegen (auch wenn das mit den Indizes teilweise recht fummelig ist). Ich empfehle, die Richtigkeit mit dem Beispiel-Input von HackerRank zu debuggen.

//-------------------------- Helper functions (lagged fibonacci) -----------------------------

void fill_rows() {
    rows.resize(N);
    for (int i = 0; i < N; i++) rows[i].resize(N);

    ll actual_f = 0, actual_g = 0;
    for (int i = 0; i < SZ; i++) {
        actual_f += fs[i];
        actual_g += gs[i];
        if (i < N * N) // add entry to rows
            rows[i / N][i % N] = as[fs[i]] + bs[gs[i]];
    }
    actual_f %= l;
    actual_g %= m;
    for (int i = 5; i < N * N; i++) {
        // add entry to rows
        rows[i / N][i % N] = as[actual_f] + bs[actual_g];
        int idx = i % 5;
        // subtract entry at i - 5 from actual
        ll tmp_f = (actual_f + (l - fs[idx])) % l;
        ll tmp_g = (actual_g + (m - gs[idx])) % m;
        // update array entry
        fs[idx] = actual_f;
        gs[idx] = actual_g;
        // calculate next values
        actual_f += tmp_f;
        actual_f %= l;
        actual_g += tmp_g;
        actual_g %= m;
    }
}

Damit ist der erste Teil erledigt und wir kommen zum Kernstück, dem Auffinden der maximal subarray sum für alle quadratischen Untergrids, die von links oben nach rechts unten wachsen sollen. Klar ist, dass wir einen Algorithmus finden müssen, der in \\( \mathcal{O}(N^2) \\) läuft, sonst drohen TLE's. Beim Durchlaufen der ganzen Subarrays müssen wir also immer um ein Element (bei den Antidiagonalen sogar um zwei Elemente) erweitern und dann den passenden Eintrag in "best" aktualisieren. 

In der Aufgabe [Subarray Sum Queries II][cses-problem] auf CSES ging es darum, in einem gegebenen Array (mit bis zu \\( 2\cdot 10^5 \\) Einträgen) in kürzester Zeit sehr viele Queries bezüglich der maximal subarray sum in sich ständig ändernden Indexintervallen zu beantworten. Ein kleiner Unterschied bestand darin, dass dort der "leere" subarray erlaubt war, d.h. die Antwort war immer mindestens null. 

Das struct "Sub" (Abkürzung für Subarray) habe ich komplett von dort übernommen. Interessant ist hier vor allem, wie man die Lösung bei der Vereinigung von zwei Subs findet und welche Kenngrößen man dazu "mitführen" muss. Dies bewerkstelligt die merge()-Funktion (die wir in dieser Allgemeinheit hier gar nicht benötigen, weil wir immer nur einzelne Elemente reinmergen, wir schießen also ein bißchen mit Kanonen auf Spatzen, aber das passt schon):

{% highlight c++ %}

//---------------------------------- Helper structs -------------------------------------------

// taken from CSES - Subarray Sum Queries II
struct Sub { // bulk sum, prefix, suffix and result
    ll blk, pre, suf, res;
};

inline Sub make_sub(ll v) {
    return {v, v, v, v};
}
// this is the decisive function to merge two subarrays
// in this problem we only need it to merge single elements from left or right
inline Sub merge(const Sub& L, const Sub& R) {
    Sub res;
    res.blk  = L.blk + R.blk;
    res.pre = max(L.pre, L.blk + R.pre);
    res.suf = max(R.suf, R.blk + L.suf);
    res.res = max(max(L.res, R.res), L.suf + R.pre);
    return res;
}

{% endhighlight %}

So, dann fangen wir doch mal mit etwas Einfachem an. Nehmen wir mal die Hauptdiagonale des Grids. Starten wir mit dem ersten Element und updaten best[0] mit dem result des Subs. Dann mergen wir das zweite Element in das Sub und updaten als nächstes best[1] und immer so weiter. Das sieht dann so aus:

{% highlight c++ %}

//-------------------------- Helper functions (update best values) ---------------------------

// let's begin with something easy
void update_with_main_diag() {
    Sub actual = {0, MIN_INF, MIN_INF, MIN_INF};
    for (int i = 0; i < N; i++) {
        actual = merge(actual, make_sub(main_diag[i]));
        best[i] = max(best[i], actual.res);
    }
}

{% endhighlight %}

Da das updaten bei den beiden Grids rows und cols absolut gleich abläuft, schreiben wir eine Funktion update_with_grid(). Jetzt muss man schon ein wenig aufpassen, dass man erst anfängt best zu aktualisieren, wenn die entsprechende Zeile oder Spalte "ins Visier" des zugehörigen Quadrats kommt. Hier die Funktion dazu: 

{% highlight c++ %}

// symmetric for rows and cols
void update_with_grid(const vector<vll>& grid) {
    for (int i = 0; i < N; i++) {
        Sub actual = {0, MIN_INF, MIN_INF, MIN_INF};
        for (int j = 0; j < N; j++) {
            actual = merge(actual, make_sub(grid[i][j]));
            // only comes into play for j >= i
            if (j >= i) best[j] = max(best[j], actual.res);
        }
    }
}

{% endhighlight %}

Wie man mit den Diagonalen updatet, kann man sich selbst überlegen. Am Härtesten ist das Updaten mit den Antidiagonalen. Man muss (abhängig von der Parität der Mächtigkeit der Antidiagonale) in die Mitte springen, dann vorne ein Element und hinten ein Element reinmergen und schlußendlich best an der richtigen Stelle aktualisieren. Das ist gar nicht so einfach, weil ab dem Index \\( N - 1 \\) die Mächtigkeit der Antidiagonalen wieder abnimmt und man daher verschiedene Variablen für die "Mitte" und den Index, an dem man updatet einführen muss. Hier habe ich recht lange schrauben müssen, bis es so lief wie ich wollte. Ein Out-of-Bounds-Zugriff (mit entsprechenden Riesenzahlen im Output) brachte mich dann auf den Bug, den ich eingebaut hatte, weil ich partout alle Antidiagonalen am Stück abarbeiten wollte. 

Ferner muss man bei den Antidiagonalen darauf achten, dass die "höheren" best-Einträge ebenfalls aktualisiert werden müssen, wenn die Antidiagonale komplett "voll" ist (das hatte ich bereits von Anfang an berücksichtigt, so schlau war ich immerhin).

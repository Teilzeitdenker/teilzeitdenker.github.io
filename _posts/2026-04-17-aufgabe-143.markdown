Gottseidank habe ich Zugang zum Forum der [ursprünglichen Aufgabe][original], denn dort gibt es ein schönes PDF zum Herunterladen, in dem eigentlich alle mathematischen Zusammenhänge erläutert werden, die man braucht, um ein ausreichend effizientes Programm zu schreiben. Daher werde ich mir sparen, hier weitere Formeln aufzuschreiben, das ist ja bereits alles geTeXt. 

Man musste nur ein wenig aufpassen, denn die HackerRank-Version fragte nach ganz anderem Output als das Original. So sollte man hier die ganzzahligen Tripel \\( (a, b, c) \\) mit \\( a \leq b \leq c \\) für jedes Toricelli-Dreieck geordnet ausgeben, während dort noch die unterschiedlichen (distinct!) \\( r + p + q \\) aufsummiert werden mussten. Definieren wir also mal ein struct für unsere Lösungs-Tripel (das uns später auch in anderer Weise dienlich sein kann): 

{% highlight c++ %}

using ull = unsigned long long;
using ui = unsigned int;
using vll = vector<ull>;

//---------------------------------- Helper structs --------------------------------------------

struct Solution {
    ull a, b, c;
    // sort ascendingly on initialization
    Solution(ull a_, ull b_, ull c_) : a(a_), b(b_), c(c_) { sort(); }
    void sort() {
        if (a > b) swap(a, b);
        if (b > c) swap(b, c);
        if (a > b) swap(a, b);
    }
    // needed for sorting a vector of solutions 
    bool operator<(const Solution& other) const { 
        // tuples will be lexicographically sorted, so use std::tie here
        return tie(a, b, c) < tie(other.a, other.b, other.c);
    }
    // output
    void print() {
        cout << a << " " << b << " " << c << endl;
    }
};

Im Folgenden werden wir genauso vorgehen, wie es im PDF beschrieben ist. Weil das maximale \\( N \\) bei HackerRank jedoch etwa 3mal so hoch angesetzt war (\\( 4\cdot 10^5 \\) statt nur \\( 1,2 \cdot 10^5 \\)), müssen wir schauen, dass wir alles so effizient wie möglich gestalten und jegliche überflüssige "Arbeit" vermeiden (insbesondere immer so früh wie möglich die Schleifen breaken!). 

Außerdem wollen wir für die inneren Schenkel festlegen, dass immer \\( r \leq p \leq q \\) gelten soll. Das bedeutet dann aber auch, dass (anders als in der Abbildung zur Aufgabe) meine inneren Dreiecke von den Seitentripeln \\( (r, p, a), (r, q, b) \\) und \\( (p, q, c) \\) begrenzt werden. Arbeiten wir uns mal vom kleinsten direkt zum größten Dreieck vor: 

1. Erstelle alle primitiven Tripel \\( (r, p, a) \\) von ganzzahligen Seiten eines 120°-Dreiecks (verläuft analog zu den primitiven pythagoräischen Tripeln, aber mit etwas anderen Formeln)
2. Erzeuge nun ALLE Tripel (mit Hilfe eines Faktors \\( k \\)) und sortiere die entstehenden Schenkellängen (legs) in eine Map-Struktur ein (die längste Seite \\( a \\) schmeißen wir weg), wobei wir den kürzeren Schenkel \\( r \\) als Key verwenden wollen und ein Vektor aus möglichen Schenkellängen \\( p \\) den zugehörigen Value darstellt
3. Gehe diese Map nun nach Keys durch. Sortiere jeden Vektor erstmal aufsteigend, dedupliziere ihn, und untersuche dann die darin enthaltenen Schenkellängen paarweise darauf, ob sie zusammen wieder als Schenkellängen in einem 120°-Dreieck \\( (p, q, c) \\) durchgehen würden (dazu muss man den Kosinussatz verwenden und überprüfen, ob eine Quadratzahl herauskommt) 
4. Falls ja haben wir eine zulässige Lösung \\( (r, p, q) \\) gefunden. Um die alte Projekt-Euler-Aufgabe zu lösen, fügen wir sie mal in ein Set ein. Praktisch ist, dass wir für dieses Set von \\( (r, p, q) \\)-Tripeln einfach das obige struct wiederverwenden können. 
5. Rechne für jedes Tripel mit Hilfe des Kosinussatzes die Zahlen \\( (a, b, c) \\) aus und füge sie in einen Lösungsvektor ein
6. Sortiere die Lösungen und schreibe alle Elemente auf stdout

Die ersten zwei Schritte sind in der Funktion fillMapOfLegs() zusammengefasst. Die Schritte 3 bis 5 in der Funktion getSolutions(). Da wir bereits in Aufgabe 141 ganzzahlige Wurzeln gezogen haben, recyceln wir die entsprechenden Funktionen hier. Allerdings erweitern wir sie mit der Funktion isSquare(). Hier der Code mit Auslassungen (und Hinweisen in den Kommentaren):

void getSolutions(ull N) {
    for (auto [r, vec] : mapOfLegs) {
        if (vec.size() == 1) continue; // no pairs
        sort(vec.begin(), vec.end());
        vec.erase(unique(vec.begin(), vec.end()), vec.end());

            for (int j = i + 1; j < (int)vec.size(); j++) {
